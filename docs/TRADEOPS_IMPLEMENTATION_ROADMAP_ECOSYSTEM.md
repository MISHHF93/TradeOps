# Implementation Roadmap — Ecosystem-First COS

## Phase A — Foundation (largely present)

* [x] CommerceCase process spine  
* [x] Process-based navigation  
* [x] Fixture vertical slice  
* [x] SaaS Stripe path  
* [x] Channel payment intelligence foundations  
* [x] Business capability framework + partner/graph APIs  

## Phase B — First live ecosystem edge (next)

* [ ] Shopify GraphQL connector package + capability ads  
* [ ] Durable Shopify webhooks (HMAC → receipt → queue → normalize)  
* [ ] Case sync from live orders/listings  
* [ ] Empty-state + next-action coverage on all stage pages  

## Phase C — Multichannel depth

* [ ] Amazon SP-API / eBay adapters (capability-first)  
* [ ] Supplier adapters beyond fixtures  
* [ ] Logistics tracking connector  
* [ ] Normalization package productized  

## Phase D — Intelligence density

* [ ] Proactive recommendation jobs (not only on-demand scan)  
* [ ] Learning memory: supplier reliability + seasonal patterns  
* [ ] Deeper knowledge-graph queries  
* [ ] Executive insights capability wired to reporting  

## Phase E — Enterprise

* [ ] SSO  
* [ ] API keys  
* [ ] Approval chains  
* [ ] Compliance reporting packs  

## Explicit non-roadmap (primary product)

* Pooled investment  
* Client capital custody as GTM  
* Competing storefront  

## Definition of ecosystem success

A merchant can connect real systems, run Discover→Learn on **live** data where authorized, with AI selecting connectors by **business capability**, every empty state teaching the next process step, and partner metrics remaining honest.
